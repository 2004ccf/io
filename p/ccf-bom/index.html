BOM 是什么？BOM 表制作 —— 从第一性原理分析

BOM = Bill of Materials，中文常称 物料清单、材料明细表、零件表。
本文从「产品究竟是什么」出发，说明 BOM 的本质、BOM 表是什么、以及如何制作。

【助理工程师必修】什么是BOM表？有什么作用？看完直接拿捏_哔哩哔哩_bilibili

一、第一性原理：产品由什么构成？

1.1 最底层的问题

任何可交付的 实物产品（手机、机床、房屋机电包、一杯装好的咖啡套餐），从第一性原理都可以拆成三个问题：

问题本质若答不清会怎样
由什么组成？零件、组件、原材料、标准件买错料、漏装、多装
各要多少？数量、单位（个、米、千克）库存错乱、成本算错
如何组织？层级、装配顺序、替代关系产线停工、返工

BOM 存在的唯一根本原因： 人类要把「脑子里的产品」变成 多人、多部门、多工厂可一致执行 的清单，且能随设计变更 可追踪地更新。

没有 BOM，就只有：

  • 工程师脑中的结构；
  • 图纸上散落的标注；
  • 采购口头确认；

—— 信息一传递就 失真、滞后、不可审计。

1.2 「图」与「表」不是同一类真理

结合机械/制造语境（与 AutoCAD、SolidWorks、Creo 等文档呼应）：

载体第一性保存的是什么典型读者
图纸 / 3D 模型几何形状、公差、装配关系（长什么样、怎么装）设计、工艺、质检
BOM 表物料身份、数量、层级、来源（用什么、用多少、谁供）采购、计划、仓库、成本、生产
  • 图纸回答：形状与配合。
  • BOM 回答：构成与数量。

二者必须 关联（通过零件号、模型属性），但 不能互相替代：
只看图可以不写清「螺栓要 48 还是 52 个」；只看 BOM 也不知道「这个支架要弯成什么角度」。

第一性结论： BOM 是产品的 组成真理（composition truth）；BOM 表是这种真理的 表格化、可交换、可计算 的表示。

二、什么是 BOM（物料清单）？

2.1 定义（去掉术语）

BOM = 一份 有结构的、带数量的物料目录，描述：

  1. 做一个产品（或交付一个装配体）需要哪些物料；
  2. 每种物料要多少；
  3. 它们之间 父子层级（谁装在谁里面）；
  4. （在完整体系里）还包含 编号、版本、来源、生效条件 等元数据。

可以把它想象成：

  • 乐高说明书里的 零件编号 + 数量表；
  • 菜谱里的 配料表 + 用量；
  • 但工业 BOM 还要管 变更、替代、多工厂、多配置。

2.2 BOM 不是「一张 Excel」

Excel 只是 载体。BOM 作为 概念 可以存在于：

  • PLM/ERP 数据库；
  • CAD 装配树导出的表；
  • 纸质明细表；
  • 与工艺路线绑定的 制造 BOM。

第一性： BOM = 数据模型 + 规则；表 = 视图之一。

2.3 常见 BOM 类型（按「谁、何时、为何」分）

同一产品在不同阶段，需要的物料视角不同，因此业界分出多种 BOM（不是炫技，是 责任界面不同）：

类型英文常称第一性：为谁服务典型内容侧重
设计 BOMEBOM / Engineering BOM设计研发设计视角的零件树、设计件号、设计状态
制造 BOMMBOM / Manufacturing BOM车间、工艺含工艺件、虚拟件、按工序展开的投料
工艺 BOMPBOM工艺工程师工序、工装、辅料与半成品
采购 BOM常由 EBOM/MBOM 衍生采购外购件、供应商、采购单位
销售 / 配置 BOMSBOM、Configurable BOM销售、选配可选配置、特征件、规则（「选装天窗则多 3 个零件」）
服务 BOMService BOM售后维修可更换单元、维修包、备件

关键： 它们往往是 同一产品在不同生命周期阶段的「不同真理切片」，需要 映射与同步，而不是简单复制粘贴。

三、什么是 BOM 表？

3.1 第一性定义

BOM 表 = 把 BOM 的层级与属性，用 行和列 展开，使人或系统能：

  • 检索（按料号、名称查）；
  • 汇总（总数量、总成本）；
  • 比对（版本 A vs 版本 B）；
  • 导入导出（ERP、MES、Excel）。

每一 行 通常对应一个 物料项（line item） 在某个 层级 上的一次出现；
列 承载属性（料号、名称、数量、单位、材料、备注等）。

3.2 层级：单层 vs 多层

形式结构适用
单层 BOM只列「直接子件」，不展开子装配内部简单产品、采购包
多层 / Indented BOM树形缩进或层级号（1、1.1、1.1.2）装配体、整机

第一性： 层级编码解决的是 「数量要在哪一层乘、在哪一层汇总」 —— 父件数量 × 子件单耗 = 该分支总需求。

3.3 BOM 表常见列（列 = 可计算的信息维度）

列名（示例）第一性含义
层级 / 序号在树中的位置
零件号 / 物料编码全球唯一身份（最重要）
名称 / 描述人类可读说明
数量每单位父件需要的子件数
单位个、套、m、kg…
材料 / 规格采购与检验依据
零件类型自制 / 外购 / 标准件
版本 / 状态Released、Working…
参考号 / 图号与几何文档挂钩
供应商采购维度
备注替代、生效、特殊工艺

列不是越多越好，而是 下游流程「做决策所需的最小完整集」。

四、BOM 表制作：从第一性到实践

4.1 制作的本质动作

制作 BOM 表 = 完成三件事：

  1. 枚举（Enumerate） — 不遗漏、不重复定义物料身份；
  2. 定량（Quantify） — 每个父–子关系上的数量正确；
  3. 结构化（Structure） — 层级反映真实装配/制造逻辑。

任何工具（手写、Excel、SolidWorks、Creo+Windchill、ERP）都只是这三件事的 自动化程度不同。

第一性建议：

  • 零件号（Item Number） 应在 设计早期 固定规则（前缀、版本位），避免后期全盘重命名。
  • CAD 自动 BOM 可靠前提是：装配里每个实体 都有唯一零件号、正确配合、无重复实例误合并。
  • EBOM → MBOM 必须有 工艺负责人 参与，不能指望一键永远正确。

4.3 典型制作路径

路径 1：Excel / 表格（小团队、样机）

步骤动作
1定层级规则（如 1、1.1、1.2）
2建主数据表（料号、名称、单位不可随意改）
3在 BOM 表中只填「料号 + 单耗」，名称用查找引用
4用公式汇总总用量（注意父件数量级联）
5版本列 + 变更说明

优点： 快、灵活。
缺点： 与 3D/PLM 脱节时，设计变更后 表最容易过期。

路径 2：3D CAD 装配导出（SolidWorks、Creo、NX 等）

步骤动作
1装配体中为每个零件维护 自定义属性（料号、材料、质量等级）
2配置/族表处理可选件（否则 BOM 会漏配或重复）
3使用 插入 → BOM / 报表 生成工程图 BOM 或导出 Excel
4区分 工程图明细表（带图号、仅 released 视图）与 完整 EBOM 导出
5人工审核：标准件库、采购件、虚拟件、封套件

第一性： CAD BOM = 装配树的投影；树错了，表一定错。

路径 3：PLM/ERP 为主（中大型企业）

步骤动作
1在 PLM（如 Windchill、3DEXPERIENCE）中 Part 对象 为真理
2CAD 模型挂到 Part 上，检入时更新结构
3变更流程（ECO）驱动 BOM 版本跃迁
4MBOM 在 ERP/MES 中由工艺维护
5BOM 表 = 系统 报表视图，而非私下维护的孤表

第一性： 当组织变大，BOM 真理必须从「某人的 Excel」迁到 受控对象 + 流程。

4.4 数量计算的底层逻辑（最易错）

对多层 BOM，某一子件 总需求 概念上为：

1

总数量 = ∏（从根到该节点的各层「每父件所需子件数」）

例： 整机 2 台 → 每台需支架组件 1 套 → 每套支架要螺栓 4 个
→ 螺栓总需求 = 2 × 1 × 4 = 8。

常见错误：

  • 只在最底层写数量，忘记乘父级；
  • 把「套件」当一件却未展开子件；
  • 同一料号在树中多处出现未 汇总（Where Used / Roll-up）。

4.5 标准件、外购件、虚拟件

类型第一性BOM 表处理
标准件行业/catalog 已有定义（螺栓 GB×）用标准料号，通常不画全 3D 细节
外购件供应商定义形状料号 + 供应商 + 采购单位
虚拟件组织用、不实物库存（如「前模块」）MBOM 可有，领料时展开到子件
制造件自制加工料号关联工艺路线与图纸

联动规则（第一性）：

  • 改几何 不一定 改数量（例如只改倒角）；
  • 改结构 一定 要动 BOM（增删子件、改单耗）；
  • 料号 一旦发布给采购，改名/合并要有 变更单，不能静默改 Excel。

六、质量检查：一张 BOM 表是否「成立」

从第一性原理，合格的 BOM 表应满足：

检查项标准
完整性能装出/生产出声明配置，无「神秘零件」
唯一性一料一号（或一号多版本有生效规则）
可追溯能回答「这个料号在哪用过」
数量闭环汇总与手算、与 CAD 质量属性抽样一致
版本一致表头版本 = 图纸/模型 Release 版本
责任清晰谁维护 EBOM、谁维护 MBOM 有定义

七、与软件工具的对应（便于落地）

场景常见做法
纯 2D 项目AutoCAD 无原生「装配 BOM」；常 手建 Excel 或借助块表、第三方工具
3D 装配SolidWorks / Creo 由装配自动生成 BOM 表，工程图插入明细栏
变更管控PLM 中 Part/BOM 版本 + ECO；Windchill、Teamcenter 等
生产执行ERP（SAP、用友等）读 MBOM 生成工单与领料

记住： 工具生成的是 草稿真理，发布前仍需 工艺/采购/质量 按角色审核。

八、总结（第一性压缩成三句）

  1. BOM 回答：「做成这个产品，需要什么料、各要多少、什么结构关系」—— 是 组成真理，不是几何真理。
  2. BOM 表 是 BOM 的 表格视图，服务于汇总、交换、导入 ERP；层级与数量规则是核心，不是排版好看。
  3. 制作 BOM 表 = 枚举 + 定 Quantify + 结构化，最佳实践是让 料号与 3D/PLM 同源，并通过 变更流程 防止私下表格与现场脱节。

九、延伸阅读（与本目录其它笔记的关系）

  • 机械 3D 定义（SolidWorks、Creo）天然产生 装配树，是 EBOM 的最佳来源之一。
  • AutoCAD 二维 项目往往需 单独维护 BOM 或与 PDM 弱联动，更依赖纪律与模板。
  • 大型集团 EBOM / MBOM 分离 是组织分工的第一性结果，不是软件 bug。
