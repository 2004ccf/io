PID控制-Give us PID controllers and we can control the world 论文笔记

1. 本次内容摘要

1.1 论文基本信息

  • 论文题目：Give us PID controllers and we can control the world
  • 出处：IFAC PapersOnLine 58-7 (2024) 103–108
  • 作者：Tore Häglund（隆德大学） & José Luis Guzmán（阿尔梅里亚大学）
  • 相关推文：微信公众号《PID控制世界？》转发了这篇论文，用一句话概括其核心观点——“给我一个 PID 控制器，我能控制整个世界”。

1.2 核心立场（作者想证明什么）

  • PID 控制是自动控制领域最重要、最有影响力的发明之一，对工业和社会的贡献可以与电、蒸汽机等经典发明相提并论。
  • 在众多现代控制算法（MPC、鲁棒控制、自适应控制、智能控制等）出现之后，PID 依然是工业界绝对主角，覆盖超过 90% 的工业控制回路。
  • PID 不应被视为“过时技术”，而应被视为自动控制领域的**“形象大使 / 基础构件”**，未来很长时间里仍会是绝大多数单回路控制问题的首选。
  • PID 与 MPC 等高级算法是互补、分层的关系，而不是“你死我活”的竞争关系。

1.3 历史与影响：从自动舵到“无处不在”

  • 1922 年，Minorsky 在为美国海军设计自动舵系统时提出 PID 控制原型。
  • 随后几十年间，PID 在**过程工业（化工、冶金、造纸等）**中成为主流，之后扩展到：
    • 能源系统（发电、输电、配电）
    • 制造和过程控制
    • 日常设备：汽车巡航控制、空调、烤箱、微波炉、无人机、电动自行车、电梯、平衡车等
  • 如今：
    • >90% 工业控制回路基于 PI/PID；
    • 几乎所有高校和技术学院在教授反馈控制时都以 PID 为标准范例；
    • 因此，PID 已经从一个算法，变成了一种基础设施级别的“隐形技术”。

1.4 “韧性”（Resilience）：为什么 PID 百年不过时

作者用“Resilience（韧性、适应力）”来形容 PID 的生命力，原因主要有三点：

  1. 适用范围极广
    • 绝大部分单回路控制问题（温度、压力、流量、转速、位移、液位等）都可以用 PID 得到“够好甚至接近最优”的效果。
  2. 现代 PID 远不止 P、I、D 三项
    • 现代工业 PID 控制器通常内置：信号滤波、前馈通道、防积分饱和（anti-windup）、平滑切换（bumpless transfer）、限幅与模式切换等。
    • 也就是说，今天的 PID 控制器本身就是一个小型控制平台，能适配复杂工况和工程需求。
  3. 简单结构 + 接近最优的性能
    • 多项研究表明，在常见工业过程（尤其是带时滞的一阶/二阶系统）上，经过合理整定的 PID，其闭环性能可非常接近更复杂的最优控制算法。
    • 在综合考虑实现成本、维护难度和鲁棒性之后，PID 往往是性价比最高的选项。

1.5 PID 与 MPC 等高级控制算法的关系

  • 现实工程中常见的结构是分层控制：
    • 上层：MPC 或其他优化控制 → 负责多变量协调、约束处理和经济优化，输出“最优设定值 / 轨迹”；
    • 下层：PI/PID 控制回路 → 负责快速反馈、局部稳定和抗扰。
  • 论文强调：
    • 不应再用“谁替代谁”的思路比较 PID 和 MPC，而应把它们视为不同层级的合作伙伴；
    • 在高层用 MPC 做优化时，底层回路绝大多数仍然会是 PID，因为 PID 足够快、足够稳、实现简单且工程师普遍能掌握。
  • 结论：
    • PID 是分层控制架构中的“基层执行核心”；
    • MPC 等算法更适合在上层做优化与协调，两者是互补而非对立。

1.6 单回路场景下的结论：PID 仍是默认首选

综合大量文献与工业经验，作者认为在单回路控制场景下：

  • 稳定过程、带一定时滞的过程、积分或弱不稳定过程等，都有成熟的 PID 设计和整定方法（IMC 思路、Ziegler–Nichols 改进规则、频域设计等）。
  • 通过合适的整定，可以在超调、响应速度、稳态误差、控制信号平滑度、鲁棒性和抗扰能力之间取得良好折衷。
  • 加上前馈、滤波、防风up和限幅逻辑后，PID 完全可以满足大多数工业现场对性能 + 安全 + 成本的综合要求。
  • 因此，在“单回路 + 工程可实施 + 维护可接受”的现实约束下，PID 在现在和可预见的未来，仍是最佳默认选择。

1.7 作者对未来的建议与呼吁

论文最后，作者发出几项呼吁：

  • 承认 PID 的历史地位和现实价值：
    它是少数被持续研究超过 100 年的控制技术之一，对工业和日常生活影响巨大，不应被简单贴上“落伍”标签。
  • 在 PID 上继续做研究很有意义：
    • 开发更系统、更易用的整定方法与工程工具，让一线工程师少踩坑；
    • 深入研究 PID 在分层控制、模块化控制中的最佳用法，与 MPC 等算法的协同方式；
    • 在教学中强调 PID 不只是一个公式，而是一整套工程实践体系。
  • 呼吁学界重视工程现实：
    考虑到 PID 在工业中的统治地位，在 PID 上每前进一步，其社会价值往往大于在冷门新算法上前进一大步。

2. 问答解惑

Q1: 现在你通俗易懂告诉我PID是什么东西，生活中哪里可以看到

精炼问题：用最简单的方式解释 PID 控制器的本质，并列举生活中常见的 PID 应用场景。

回答：

PID 是什么？用一句话说

PID 就是一个“自动纠错系统”，它的工作就是：不停地看着“目标值”和“实际值”差多少，然后自动调整，让实际值尽快追上目标值。

用开车比喻 PID

想象你在高速上开车，想保持 120 公里/小时：

  • P（比例）：你发现现在只有 100 公里/小时，差 20，就立刻踩油门。差得越多，踩得越狠。
  • I（积分）：如果一直差一点（比如总是 118，到不了 120），你会越来越“焦虑”，持续加大油门，直到完全追上。
  • D（微分）：你发现速度上升太快（比如从 100 突然跳到 130），就提前松油门，避免超速。

PID 就是把这三个动作组合起来，自动帮你把速度稳定在 120。

生活中哪里可以看到 PID？

1. 空调（温度控制）

  • 你设定 26℃，空调里的 PID 控制器会：
    • 检测室内温度
    • 如果低于 26℃，就制冷；如果高于 26℃，就制热
    • 不停微调，让温度稳定在 26℃ 附近

2. 电热水壶（温度控制）

  • 设定 100℃ 烧水
  • PID 控制器检测水温，快到 100℃ 时慢慢减小功率，避免烧过头

3. 汽车巡航定速（速度控制）

  • 设定 80 公里/小时
  • PID 控制器自动调节油门，保持速度稳定，上坡时多给油，下坡时少给油

4. 平衡车 / 电动滑板车（位置/角度控制）

  • 你身体前倾，车就往前加速；后仰就减速
  • 背后是 PID 控制器在检测你的重心角度，自动调整电机转速

5. 无人机（高度/姿态控制）

  • 你让它悬停在 10 米高度
  • PID 控制器检测高度传感器，自动调节螺旋桨转速，保持稳定

6. 电梯（位置控制）

  • 电梯要停在 5 楼，快到 5 楼时 PID 控制器会：
    • 先减速
    • 精确停在楼层位置
    • 不会冲过头或停不到位

7. 烤箱 / 微波炉（温度控制）

  • 设定 180℃ 烤蛋糕
  • PID 控制器检测温度，自动开关加热元件，保持温度稳定

8. 洗衣机（转速控制）

  • 设定 800 转/分钟甩干
  • PID 控制器检测电机转速，自动调节电压/频率，保持转速稳定

9. 恒温花洒（水温控制）

  • 设定 38℃ 水温
  • PID 控制器检测水温，自动调节冷热水比例

10. 手机充电（电压/电流控制）

  • 快充时，PID 控制器检测电池电压和电流，自动调节充电功率，避免过充

为什么 PID 这么常见？

因为几乎所有“需要保持某个值稳定”的场景，都可以用 PID：

  • 温度 → PID
  • 速度 → PID
  • 位置 → PID
  • 压力 → PID
  • 液位 → PID
  • 流量 → PID

PID 就像是一个“万能稳定器”，只要你想让某个物理量稳定在目标值，PID 基本都能搞定。

总结

  • PID = 自动纠错系统
  • 核心工作：检测误差 → 自动调整 → 让实际值追上目标值
  • 生活应用：空调、热水壶、汽车巡航、平衡车、无人机、电梯、烤箱、洗衣机、恒温花洒、手机充电等
  • 为什么常见：因为生活中太多东西需要“保持稳定”，PID 就是最简单、最可靠的稳定方法

💡 延伸思考：

  • PID 的三个参数（P、I、D）分别控制什么？如果调不好会有什么后果？（比如空调温度一直波动、平衡车容易摔倒）
  • 为什么有些设备（比如老式电风扇）不需要 PID，而现代智能设备都需要？
  • 你在 FMTP 设备里，哪些地方可能用到了 PID 控制？（比如伺服电机位置控制、温度控制、压力控制等）

Q2: 所有PID是算法，存在于软件里是吗

精炼问题：PID 是否都是软件算法？是否存在硬件实现？现代 PID 的实现方式有哪些？

回答：

简短回答

不是的。PID 既可以是软件算法，也可以是硬件实现，还可以是软硬件混合。 现代大多数 PID 控制器是软件算法，但历史上和某些特殊场景下也有纯硬件实现。

详细解释：PID 的三种实现方式

1. 软件实现（最常见，现代主流）

特点：PID 算法写在程序代码里，运行在 CPU/微控制器上。

常见场景：

  • PLC（可编程逻辑控制器）：比如你在博图（TIA Portal）里写的 PID 功能块，就是软件算法
  • 单片机/嵌入式系统：比如平衡车、无人机里的 PID 控制代码
  • 工控机/PC：比如用 C/C++、Python、MATLAB 写的 PID 控制程序
  • DCS（分布式控制系统）：工业控制系统的 PID 模块，本质也是软件

优点：

  • ✅ 灵活：可以随时修改参数、调整算法
  • ✅ 成本低：不需要专门的硬件
  • ✅ 功能强：可以加入滤波、防风up、前馈等复杂功能
  • ✅ 易维护：升级、调试都方便

例子：

  • 你在博图里调用 PID_Compact 功能块 → 这是软件算法
  • 空调里的微控制器运行 PID 程序 → 这是软件算法
  • 汽车 ECU（发动机控制单元）里的巡航控制 → 这是软件算法

2. 硬件实现（传统方式，现在较少）

特点：用模拟电路（电阻、电容、运算放大器等）直接实现 PID 计算，不需要 CPU。

常见场景：

  • 老式模拟控制器：比如 1970-1990 年代的工业控制器
  • 专用 PID 芯片：某些特殊应用场景
  • 纯模拟系统：比如一些简单的温度控制器

工作原理：

  • P（比例）：用运算放大器做比例放大
  • I（积分）：用电容 + 运算放大器做积分运算
  • D（微分）：用电感/电容 + 运算放大器做微分运算

优点：

  • ✅ 响应极快：不需要 CPU 计算，纯电路响应
  • ✅ 实时性好：不受软件扫描周期影响
  • ✅ 可靠性高：没有软件 bug 风险

缺点：

  • ❌ 不灵活：改参数要换电阻/电容，很麻烦
  • ❌ 成本高：需要专门的模拟电路
  • ❌ 功能单一：很难加入复杂功能

例子：

  • 老式温度控制器（用旋钮调 P、I、D 参数，背后是模拟电路）
  • 某些高速响应系统（比如激光器功率控制，需要微秒级响应）

3. 软硬件混合（现代高端系统）

特点：硬件加速 + 软件控制，把 PID 的核心计算放在专用硬件（FPGA、DSP）上，但参数调整、逻辑控制用软件。

常见场景：

  • 高速运动控制：比如高精度 CNC 机床、机器人关节控制
  • 实时性要求极高的系统：比如航空航天、医疗设备
  • 多轴同步控制：需要多个 PID 同时高速运行

优点：

  • ✅ 速度快：硬件计算，响应快
  • ✅ 灵活：软件调整参数和逻辑
  • ✅ 精度高：专用硬件保证计算精度

例子：

  • 伺服驱动器里的 PID 控制（DSP 芯片 + 软件参数调整）
  • 高精度机器人控制器（FPGA 实现 PID 计算，上位机软件调整参数）

现代趋势：软件实现占主流

为什么现在大多数 PID 都是软件实现？

  1. 成本低：一个几块钱的单片机就能跑 PID 算法
  2. 灵活性强：可以随时改参数、加功能、远程调试
  3. 功能丰富：可以加入滤波、防风up、自适应、前馈等复杂功能
  4. 易维护：升级、备份、远程监控都方便
  5. 性能足够：现代 CPU/微控制器速度很快，软件 PID 的响应速度已经能满足 99% 的应用需求

什么时候还需要硬件实现？

  • 响应时间要求极快（微秒级，比如激光器、高频电源）
  • 系统不允许有任何软件延迟（比如某些安全关键系统）
  • 成本敏感但性能要求不高的简单应用（某些老式控制器）

总结对比表

实现方式常见场景优点缺点现代使用率
软件实现PLC、单片机、工控机、DCS灵活、成本低、功能强、易维护受 CPU 性能限制>95%
硬件实现老式模拟控制器、高速系统响应极快、实时性好不灵活、成本高<3%
软硬件混合高速运动控制、多轴同步速度快+灵活成本较高<2%

对你学习的影响

在博图（TIA Portal）里：

  • 你调用 PID_Compact 功能块 → 这是软件算法
  • 这个算法运行在 PLC 的 CPU 上
  • 你可以通过 HMI 或程序随时修改 P、I、D 参数
  • 这就是典型的软件实现 PID

在 FMTP 设备里：

  • 伺服电机的位置控制 → 可能是软件 PID（在 PLC 里）或软硬件混合（在伺服驱动器里）
  • 温度控制 → 通常是软件 PID（在 PLC 或温控器里）
  • 压力控制 → 通常是软件 PID

💡 延伸思考：

  • 为什么现代工业控制系统（如 PLC）里的 PID 都是软件实现？软件 PID 的响应速度能满足工业需求吗？
  • 在博图里，PID 功能块是如何在 PLC 的扫描周期中执行的？会不会影响控制精度？
  • 如果遇到响应时间要求极快的场景（比如微秒级），软件 PID 还能用吗？需要什么特殊处理？

3. 内容主题划分

3.1 引言与自动控制的“隐藏技术”

  • 自动控制被称为“隐藏技术”，广泛存在于经济、生命科学、能源、交通和制造业中。
  • 反馈是自动控制的心脏，PID 是反馈在工业界最成功、最现实的实现方式。
  • 论文用“给我们 PID 控制器，我们就能控制世界”来强调 PID 的重要性。

3.2 PID 的历史与应用版图

  • 起源于 1922 年的自动舵系统；
  • 在过程工业中发展壮大，随后扩展到能源、制造和日常家电/设备；
  • 如今覆盖 >90% 工业控制回路，并成为全球高校教学反馈控制的标准算法。

3.3 PID 的演化与“韧性”

  • 从最初的 P/I/D 三项，演化为包含滤波、防风up、前馈、平滑切换等能力的现代 PID 控制平台；
  • 与各类高级控制结构（级联、多变量、分层、自适应）深度结合，成为这些结构的底层执行模块；
  • 在多种典型工业过程上，性能接近更复杂的最优控制方案，说明其结构虽简单但非常高效。

3.4 PID 与 MPC 等高级控制算法的分工

  • 分层控制架构：上层 MPC 做约束与经济优化，下层 PID 做快速反馈与抗扰；
  • 高级算法不是用来全面替代 PID，而是与 PID 协同工作；
  • 从工程可实施性、维护成本和人员技能结构看，PID 长期仍是底层控制的重要基石。

3.5 面向未来的研究方向

  • 更友好、更鲁棒的 PID 整定方法与工程工具；
  • PID 在模块化/分层控制架构中的位置与接口设计；
  • 如何在工业 4.0、智能制造背景下，把 PID 与数据驱动、AI、MPC 等新技术更好融合。
