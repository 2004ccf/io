工程公差分析是什么？为什么要做？—— 从第一性原理分析

公差（Tolerance） = 允许尺寸、形状、位置等几何量偏离 名义值（Nominal） 的 permissible 范围。
公差分析（Tolerance Analysis） = 用数学与统计方法，判断 各零件误差叠加后，产品 关键功能尺寸 是否仍落在要求范围内。

一、第一性原理：制造世界没有「绝对精确」

1.1 理想 vs 现实

设计图纸上写的 50.00 mm，在哲学上是 名义尺寸 —— 一个 目标中心，不是自然界能无限逼近的「真值」。

第一性事实：

层面现实
物理温度、振动、刀具磨损、材料去除不确定性
工艺车、铣、注塑、冲压各有能力极限（工艺能力）
测量量具本身有分辨力与误差
装配孔轴对中、垫片、拧紧顺序都会改变最终间隙

因此：每一个制造出来的尺寸都是一个随机变量（或区间变量），而不是一个数。

公差分析的第一性起点不是「怎么标 ±0.1」，而是：

承认误差必然存在，并问：在误差必然存在的前提下，产品还能不能可靠地工作？

1.2 「能工作」往往是一个 区间问题

许多功能本质上是 间隙、过盈、角度、相位 是否落在某区间：

功能第一性问法
轴在轴承中转动间隙是否 ≥ 最小润滑间隙 且 ≤ 最大晃动？
盖板密封压缩量是否在垫片有效范围内？
齿轮啮合中心距偏差是否仍在齿形包容区内？
光学对准累积偏位是否仍小于焦深预算？

公差分析 就是把「每个零件偏一点」翻译成 「装起来以后，关键量还合不合格」 的推理过程。

二、什么是公差？（标在图上的 vs 心里的）

2.1 尺寸公差

对长度、直径等：

1
2

名义尺寸 50 mm，上偏差 +0.05，下偏差 -0.03
→ 合格范围 [49.97, 50.05] mm

第一性： 公差 = 用区间代替单点，告诉制造与检验「多宽算对」。

2.2 几何公差（GD&T）

位置度、平行度、圆跳动等 —— 约束的是 形状与相对关系，而不只是某一截面的直径。

第一性： 很多功能失效来自 「尺寸都合格但方向歪了」；GD&T 把 功能约束 直接绑在基准体系上。

2.3 公差不是「越小越好」

公差越紧后果（第一性）
加工工序升级、废品率上升、节拍变慢
测量需要更高等级量具与环境
装配选型件、修配、筛选增加成本

公差是成本与风险之间的合同：
太松 → 功能/可靠性风险；太紧 → 金钱与时间浪费。

公差分析的使命之一就是：找到满足功能的最松公差组合（成本最优）。

三、什么是公差分析？

3.1 定义（去掉术语）

给定：

  1. 一个 装配关系（谁靠谁、间隙如何形成）；
  2. 各零件相关尺寸的 公差规格（区间或分布）；
  3. 一个 关键功能尺寸（Closing Dimension / Critical Characteristic）—— 如总间隙、中心距；

公差分析 计算：

当所有输入尺寸在其公差内变化时，关键尺寸的变化范围（或概率分布） 是什么？
它是否落在 功能要求 之内？

3.2 两类核心算法（第一性）

方法思想适用
极值法（Worst Case / Stack-up）所有误差同时取最不利组合（全最大或全最小）保守、样本少、安全件、法规件
统计法（RSS、蒙特卡洛等）各尺寸独立随机，大量组合或方差叠加批量生产、中心极限定理近似、成本敏感

极值法第一性：
「只要每个零件都合格，最坏情况下装在一起也不能坏」—— 零失败容忍 时的逻辑。

统计法第一性：
「所有尺寸同时处于极值的概率极低」—— 用 过程能力 + 分布 换更合理的公差分配，避免过度设计。

3.3 尺寸链（Dimension Chain）

装配体里，关键尺寸往往是若干零件尺寸的 和、差或三角函数关系：

1

关键间隙 G = A - B - C   （简化示意）

公差分析就是在 传播 各 A、B、C 的误差到 G：

  • 极值：( G_{max} = A_{max} - B_{min} - C_{min} ) 等；
  • 统计：( \sigma_G^2 \approx \sigma_A^2 + \sigma_B^2 + \sigma_C^2 )（独立、线性时 RSS）。

第一性： 公差分析 = 误差在装配拓扑上的传播计算。

四、为什么要进行公差分析？（核心章节）

许多团队以为：「图纸上标了公差，工作就完成了。」
从第一性原理看：标公差 只解决「单个零件合不合格」；公差分析 解决「合不合格的零件装在一起，产品还能不能正常干活」。
后者才是客户付钱要买的东西。

4.1 根本矛盾：合格零件 ≠ 合格产品

这是整章的逻辑起点。

1
2
3
4
5

零件 1 ∈ 公差带  ✓
零件 2 ∈ 公差带  ✓
零件 3 ∈ 公差带  ✓
        ↓ 装配
关键间隙 G  ∉ 功能允许范围  ✗   ← 没有公差分析时，常到试制才发现

装配把多个区间 相加、相减、形成配合。
第一性： 系统行为是 结构的函数，不是零件行为的简单列表。
因此必须进行公差分析，把 系统级问题 在设计阶段算清，而不是交给试制车间「锉到能装为止」。

4.2 八大原因（从第一性展开）

（1）保证功能与可靠性 —— 设计存在的理由

客户买的是 密封、运转、对准、手感、安全，不是买的是「每个尺寸在带内」。

功能类型公差分析要回答什么
旋转/滑动最小间隙是否够润滑？最大间隙是否导致晃动、噪声？
密封压缩量是否在垫片/ O 圈有效窗口？
定位/光学累积偏位是否超过焦深、对准预算？
电气/热热胀后间隙是否变负（卡死）？

不做： 功能在图纸上看不出，问题延迟到 首件装配、可靠性试验、市场投诉。
要做： 在纸上/模型阶段就证明 最坏或统计意义下仍满足功能。

（2）控制成本 —— 公差是「花钱的开关」

加工精度大致 每紧一档，成本非线性上升（工序、设备、废品、节拍）。

策略第一性
无分析、一律紧公差为「心里踏实」买下全系统最贵加工
无分析、一律松公差量产失效、筛选、返修、召回
有分析只对 尺寸链上「吃预算」的少数环 收紧，其余放松

公差分析是 用数学换金钱：在满足功能的条件下，找到 最松的可行公差组合（最大材料条件、最优成本）。

（3）分配公差预算 —— 避免「环环都要 ±0.01」

功能允许的总偏差有限（一块蛋糕）。
设计有 8 个尺寸参与配合时，若凭感觉每个都标 ±0.05，可能 叠加后远超 功能需要；或某一环 单独过紧 却挤占了其它环空间。

公差分析的作用：

  • 先定 关键尺寸（Closing dimension） 的功能上下限；
  • 再 反推或分配 各环公差（等预算、按灵敏度加权等）；
  • 让采购、工艺知道 哪几个尺寸是真正「关键特性」。

（4）支持并行开发与责任清晰 —— 组织第一性

现代产品 设计、工装、供应商、质检 并行。若没有统一的尺寸链分析：

  • 甲供应商孔偏大、乙轴偏小，都在各自图纸内合格，总装却失败；
  • 无法判断应 改孔、改轴、还是改结构消除一环。

公差分析输出的是 系统契约：

「为实现间隙 G∈[0.1, 0.25]，A 负责环 1、B 负责环 2，各自上限如下。」

减少扯皮，也避免 过度依赖装配工修配（不可规模化）。

（5）评估设计变更 —— 改一处、算全局

工程变更（ECO）常只改一个孔、一张板厚。
没有分析时，凭直觉「应该没事」—— 却可能 耗尽公差预算 或引入过约束。

公差分析提供：

  • 变更前后关键尺寸分布对比；
  • 是否需同步改工艺、检具、供应商规格；
  • 变更的 风险量化（极值最坏情况 / 失效概率估计）。

（6）指导 DFM / DFA —— 从「事后修」到「结构避错」

分析常发现：不是公差标紧就能解决，而是 尺寸链太长、过约束、基准与装配顺序不一致。

此时应改结构，例如：

  • 用 定位销 + 功能面 减少环数；
  • 用 浮动螺母、自对准轴承座 吸收累积；
  • 合并零件减少接口。

第一性： 公差分析是 设计评审的定量输入，而不只是质检部门的表格。

（7）对接工艺与六西格玛 —— 纸面与现场一致

图纸公差必须 工艺能力（Cp/Cpk）可实现。
公差分析若只用 RSS，而现场 σ 很大 → 纸上合格、线上超差。

因此分析循环应是：

1

功能要求 → 公差分析（理论）→ 试制/SPC 数据 → 修正假设 → 发布量产图纸

不做分析 → 工艺不知道设计 真正在意哪一个尺寸 → 资源投错地方。

（8）合规、认证与客户证据 —— 高可靠行业刚需

汽车、航空、医疗、精密仪器等常要求：

  • 关键特性（CTQ） 清单；
  • 极值或统计叠加的 记录与评审；
  • 与 FMEA、控制计划衔接。

公差分析提供 「为何这样标」的可追溯证据，不是「老师傅经验」。

4.3 谁受益？（不做时谁最痛）

角色做公差分析的好处不做时的典型痛点
设计早期发现结构问题，少改模试制被叫去现场「锉刀会诊」
工艺知道关键尺寸，工装检具对准全检、筛选、临时放宽
采购/供应商清晰传递系统要求来料合格仍被退货
质量CTQ 明确，检验资源聚焦抓不住真正原因
商务/项目少延期、少报废量产爬坡失败、成本超支

4.4 什么时候必须做？（决策门槛）

情形建议
多零件装配形成 关键间隙/过盈必须
尺寸链 ≥ 3 环 且功能窗口窄必须
安全、法规、客户明确要求必须
高精度光学、传动、液压密封必须
单件、非关键、功能对尺寸不敏感可简化，但仍建议最小链检查
大批量量产、成本敏感必须（宜统计法 + 过程数据）

第一性： 只要 「系统功能 = 多个误差的函数」 且 「失效代价 > 分析成本」，就值得做。

4.5 不做公差分析的代价（用结果说话）

阶段可能后果
试制装不上、修配、改模一轮轮
小批量筛选工位、废品率飙升
量产Cpk 不足、停线、供应商索赔
售后泄漏、异响、磨损加速、召回
组织设计-制造对立、经验依赖个人

这些成本往往 远高于 一次系统的尺寸链计算或专用软件投入。

4.6 与「只标公差」的本质区别（再强调）

只标公差进行公差分析
回答：每个零件 单独 是否合格回答：产品 装起来 是否仍满足功能
局部、静态系统、面向装配与功能
常拷贝同类图 ±0.1有 公差预算 与依据
变更凭感觉变更可 重算、可审计

一句话：
标公差是 给制造的下限命令；公差分析是 给产品的功能保险 —— 这就是为什么要 进行 公差分析，而不只是 标注 公差。

4.7 典型失败场景（反例速查）

  1. 每个零件都合格，装配不合格 —— 误差同向累积。
  2. 关键间隙过紧 —— 量产靠筛选、修锉。
  3. 基准与装配顺序不一致 —— 测量合格、功能失败。
  4. 忽视温度、磨损、涂层厚度 —— 常温 OK、工况失败。
  5. 多供应商「各自合格」 —— 总装无系统预算。
  6. 公差过紧无工艺能力 —— 检验全红、交付延期。

公差分析的目标，是在 设计阶段 用计算或仿真 预演 上述失败，而不是在客户现场「第一次实车验证」。

5.2 公差预算（Tolerance Budget）

第一性： 功能允许的总偏差 = 一整块蛋糕；每个零件切走一块，不能切超。

例：总间隙允许 ±0.2 mm 波动
→ 分配给轴径、孔径、同轴度、板厚…
→ 若某环已用掉 80% 预算，其余环必须更紧或 改设计消除一环。

5.3 结构手段（与「算」并列）

公差分析常导向 设计改型，而不只是改数字：

手段第一性
对称结构误差正负抵消
浮动/自对准用功能面定位，减少过约束
可调垫片、偏心销吸收累积误差（装配成本换功能）
减少尺寸链环数短链比长链更可控
统一基准系测量与装配同一参考

六、极值法 vs 统计法：何时用哪个？

维度极值法统计法（RSS / Monte Carlo）
哲学最坏必发生极端组合概率低
结果偏保守更接近量产现实
样本小批量、安全关键大批量、数据充分
数据需求只需上下限最好有 σ、Cp/Cpk
风险可能过度收紧成本假设独立/正态不当会低估风险

第一性选择：

  • 问「有没有可能 只要合格件装一起就一定坏？」→ 是，先用极值。
  • 问「批量下 失效概率是否可接受？」→ 用统计 + 过程数据。

航空、医疗等常 极值 + 额外安全系数；消费电子大量用 蒙特卡洛 + 实测修正。

公差分析验证 约束组合后仍可行
Cp/Cpk工艺 实际 能稳定落在公差内的程度
六西格玛用数据把「缺陷率」与公差、变异挂钩

公差分析若脱离工艺能力： 纸上 RSS 很美，机床做不到 → 仍失效。

九、质量检查：一次公差分析是否「成立」

检查项标准
功能限明确最小/最大间隙或性能指标有数
尺寸链完整无遗漏环、方向正确
基准一致与图纸 GD&T、装配工艺一致
方法声明极值还是统计、假设是什么
与工艺对齐公差宽于或等于 achievable Cp
变更可追溯改一环重算关键尺寸

十、总结（四句话）

  1. 制造没有绝对尺寸，公差是承认现实后的 合格区间合同。
  2. 公差分析 不是标 ±号，而是计算 误差沿装配链传播后，关键功能是否仍安全 —— 是 系统风险与成本的量化。
  3. 为什么要进行公差分析：因为 合格零件 ≠ 合格产品；要为功能、成本、并行协作、变更、DFM、工艺能力与合规 提供系统级证据，而不是把风险留给试制和锉刀。
  4. 标公差 约束零件；做分析 约束产品 —— 二者缺一不可。
