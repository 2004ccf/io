西门子「博图 / 博途」PLC 软件是什么？—— 从第一性原理分析

说明： 国内口语常把西门子这套软件叫 「博图」 或 「博途」，官方中文品牌名为 SIMATIC STEP 7 / TIA Portal（全集成自动化门户）。
你桌面截图中的 TIA Portal V18 即其核心；其余图标为 仿真、人机界面运行、端口配置 等配套工具。

一、第一性原理：PLC 软件解决什么问题？

1.1 产线控制的根本问题

机械与机器人（CAD、PDPS）解决 「物怎么造、动作怎么动」；
真正让设备 按顺序、安全、可重复地运转，靠的是 电气控制与逻辑：

根本问题谁回答
什么时候开阀、何时启动电机？控制逻辑（PLC 程序）
传感器亮了下一步做什么？顺序 / 状态机 / 联锁
人怎么操作、怎么看状态？人机界面（HMI/SCADA）
程序没写好就上电会怎样？撞机、误动作、人身风险

PLC（可编程逻辑控制器） 的第一性角色：
在恶劣工业现场，用 确定的扫描周期 反复执行 逻辑 + 运算 + 通信，把 工艺意图 变成 I/O 信号。

博途 / TIA Portal 的第一性角色：
为西门子 PLC、HMI、驱动、IO 设备 提供 统一的工程、编程、仿真、下载与诊断环境 —— 即 「控制真理」的编写与交付工具。

1.2 在自动化层级中的位置

产品设计        CATIA / NX / SolidWorks     →  零件、机构
制造仿真        PDPS / DELMIA               →  工艺、机器人轨迹
控制与自动化    TIA Portal（博途）          →  PLC 逻辑、HMI、驱动配置
现场执行        PLC + 变频器 + 阀 + 传感器

第一性结论： 博途不是画机械图，而是 让机器按逻辑安全运行；与 PDPS 的 虚拟调试 可对接（同一 PLC 程序可在 PLCSIM 里先跑通）。

二、「博图 / 博途」到底是什么？

2.1 名称对照

叫法含义
博途西门子中国官方对 TIA Portal 的惯用中文名
博图业内口语，多指 同一套 TIA Portal，偶与旧版 STEP 7 混称
TIA PortalTotally Integrated Automation Portal，英文产品名
STEP 7传统名称；现 编程 S7-1200/1500 等 多在 TIA Portal 内完成
SIMATIC西门子 可编程控制器产品品牌 总称

2.2 它是什么（一句话）

TIA Portal（博途） = 西门子 一站式工业自动化工程软件平台，在同一安装里可完成：

  • PLC 编程（梯形图 LAD、语句表 STL、结构化文本 SCL、功能块图 FBD 等）
  • HMI 组态（触摸屏画面、报警、配方）
  • 驱动配置（如 SINAMICS 变频器参数）
  • 网络与硬件组态（Profinet、IO 模块、CPU 型号）
  • 仿真、下载、在线诊断

你截图中的 V18 表示 第 18 代主版本（需对应授权；高版本项目通常可向下兼容策略依官方说明为准）。

四、主要作用（能干什么）

4.1 对 PLC 工程师

  • 配置 CPU、电源、IO 模块、通信模块；
  • 编写 逻辑控制：电机启停、互锁、定时、计数、PID、安全相关逻辑（配合安全 PLC）；
  • 在线监控：强制 I/O、断点、跟踪变量、诊断缓冲区；
  • 固件升级、密码保护、版本管理（结合库与项目归档）。

4.2 对自动化产线

  • 协调 输送线、机器人、夹具、阀门 的时序（通过 Profinet、IO、通信 FB）；
  • 与 上位机、MES、SCADA 交换数据；
  • 故障 报警、复位流程、模式切换（手动/自动/维修）。

4.3 与 PDPS / 数字孪生的衔接

  • Process Simulate 虚拟调试 常通过 PLCSIM Advanced + TIA Portal 运行 真实 PLC 程序，在 3D 仿真里验证逻辑与动作同步；
  • 第一性： PDPS 管 几何与运动；博途管 逻辑与 I/O；二者在 虚拟 commissioning 阶段汇合。

4.4 典型行业

  • 产线自动化、包装机械、水处理、楼宇、物流、机床、汽车辅线等凡用 西门子 S7-1200/1500/ET200 处均常见。

五、主要特点（从第一性推导）

特点解释
全集成（TIA）PLC + HMI + 驱动尽量 一个软件、一个项目树，减少重复组态
硬件.catalog 丰富按订货号拖模块，自动校验容量与版本
多语言编程电工习惯梯形图；复杂算法用 SCL；团队可分工
仿真链条PLCSIM（基础/Advanced）降低 上电试错 风险
诊断强在线连接后可查模块状态、通信、强制表，利于维保
版本与许可分档Basic / Professional 等；功能与设备型号绑定，需正版授权
学习曲线比单片机环境重；但 工业标准工具，就业与项目面广
与西门子生态绑定同厂 PLC、HMI、驱动、工业网络；换品牌需重写工程

六、核心概念（读界面时不迷路）

概念第一性含义
项目（Project）一台设备或一条线的 全部自动化对象的容器
设备视图 / 设备配置CPU 插了哪些模块、IP、网络
PLC 程序块OB（循环）、FB/FC（功能）、DB（数据）、库复用
HMI 画面操作员看的按钮、指示灯、趋势
编译 / 下载把工程变成 CPU 可执行代码并写入
在线 / 离线是否连接真实或仿真 PLC

PLC 扫描周期第一性（简化）：
读输入 → 执行程序 → 写输出 → 循环；程序必须在 一个周期内算完，否则看门狗报警。

七、与「三菱、欧姆龙、AB」等对比（选型层面）

维度西门子 TIA（博途）
主流机型S7-1200（中小型）、S7-1500（中大型）
软件仅 TIA Portal 一条主线（旧 STEP7 经典已边缘）
优势欧洲项目、汽车、大型 OEM、与西门子驱动/仪表一体
切换成本换品牌 = 硬件 + 软件 + 人员 全换

第一性： 选博途往往因为 项目指定西门子 PLC 或 集团标准化，不是因为「语法最好看」。

八、什么时候需要学 / 用博途？

情形建议
设备用 S7-1200/1500必须学 TIA Portal
做 PLC 编程、电气设计、调试核心工具
做 PDPS 虚拟调试需会 PLCSIM Advanced + TIA 联仿
只做机械设计、无电气了解接口即可（I/O 清单、通信需求）
仅用三菱/汇川 PLC不必学博途（对应 GX Works 等）

九、学习路径（实用）

  1. 认识硬件：CPU、DI/DO、AI/AO、Profinet 模块。
  2. TIA Portal：建项目 → 添加设备 → 写 梯形图 启停互锁 → 仿真下载到 PLCSIM。
  3. 进阶：SCL、FB 封装、通信（Modbus TCP、S7 连接）、HMI 基础画面。
  4. 与仿真/产线：PLCSIM Advanced、与 PDPS/WinCC 协作（岗位相关再学）。

十、软件分享（可直接转发）

西门子博途（TIA Portal） 是西门子 PLC 和自动化设备的编程与组态软件。你电脑上的 V18 是主程序；PLCSIM Advanced 用来在电脑里假跑 PLC；WinCC Runtime 用来运行触摸屏程序；S7-PCT 用来配 IO-Link 设备。

作用： 写控制逻辑、配硬件、做 HMI、下载到控制器、调试产线。
特点： 全家桶集成、工业级诊断、和西门子 PLC 深度绑定、需授权。
不是： 机械 CAD，也不是 PDPS 工艺仿真本身，但是 产线「大脑」的开发环境。

和 PDPS 的关系：PDPS 仿真 机器人动作；博途写 何时允许机器人动；虚拟调试时两边 联调。
